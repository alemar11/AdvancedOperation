  • DelayOperation

    An AdvancedOperation that will simply wait for a given time interval, or until a specific Date. If the interval is negative, or the Date is in the past, then this operation immediately finishes.

    Note

    A DelayOperation cannot be cancelled once started running.

  • GroupOperation

    An AdvancedOperation subclass which enables a finite grouping of other operations. Use a GroupOperation to associate related operations together, thereby creating higher levels of abstractions.

    Note

    The progress report is designed to work with a serial GroupOperation: if the operations are run concurrently the progress report may be less accurate (i.e. KVO could report the same percentage multipe times, except for the final value).

    Attention

    If you add normal Operations, the progress report will ignore them, instead consider using only AdvancedOperations.
